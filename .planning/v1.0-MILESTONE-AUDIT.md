### Issue: Goals Screen PIN Disconnect

The Goals screen eye-icon PIN toggle (`authStore.isUnlocked`) is disconnected from the privacy masking system (`settingsStore.isPrivacyMode`). Entering PIN on Goals screen has no effect on displayed target names. The Settings screen toggle is the working mechanism.

**Workaround:** Users can toggle privacy via Settings screen. All masking components work correctly via that path.

**Fix:** `goals.tsx handlePrivacyToggle` should call `settingsStore.togglePrivacyMode()` in addition to or instead of `authStore` mutations.
