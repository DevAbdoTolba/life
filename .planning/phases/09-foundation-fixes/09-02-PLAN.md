<objective>
Fix the body-fill physics visualization (BUG-02) so balls actually render and fall with physics, and fix the screen wake lock issue (BUG-01) by ensuring the RAF loop stops after settlement.

Purpose: The body-fill visualization currently renders zero balls because `BodyFillCanvas` reads an empty `useRef` at mount time and never re-renders when it is populated. The RAF loop never stops even after balls settle, preventing the OS from dimming the screen. Both bugs are in the same hook and must be fixed together.

**D-07 deviation:** Decision D-07 requested moving the physics tick from JS RAF to Reanimated `useFrameCallback` for UI-thread execution. Per RESEARCH.md Pattern 2, this is infeasible: `useFrameCallback` runs its callback on the UI thread as a worklet, but `Matter.Engine.update()` is pure JS and cannot execute inside a worklet. Literal compliance would crash the app. This plan keeps the RAF loop on the JS thread (which is the correct architecture for Matter.js) and writes SharedValue positions from there. SharedValue `.value` writes are cross-thread safe, so the Skia canvas still animates at 60fps on the UI thread. The real performance wins come from pre-allocation (D-06/D-09) and RAF termination (BUG-01), not the thread model.

Output: Working body-fill physics with pre-allocated ball slots, fast-drop bounce feel, and automatic RAF termination after settlement.
</objective>
