## Animation Contract

Phase 9 touches two animation systems. Both must maintain their current visual parameters.

### Body-fill ball physics (BUG-02)

| Parameter | Current (broken) | Fixed (Phase 9) |
|-----------|-----------------|-----------------|
| Initial ball positions | Invisible (bug) | Spawn at `y = -radius` (above canvas top), drop into silhouette |
| `restitution` (bounce) | `0.3` | `0.6` — higher bounce, "fast drop + satisfying bounce" (D-05) |
| `friction` | `0.1` | `0.05` — lower surface friction, balls slide to rest more freely |
| `frictionAir` (drag) | `0.02` | `0.01` — lower air drag, faster fall |
| `gravity.y` | `1.2` | `2.0` — faster drop speed |
| RAF stops when | Never (bug) | `VELOCITY_THRESHOLD = 1.0` for all bodies + 30s hard timeout |

**Feel target (D-05):** Fast drop, 2-3 visible bounces, settle cleanly. Playful and energetic. Not slow, not gentle.

**Source:** `src/components/physics/useBodyFillPhysics.ts` lines 81 and 119-123; CONTEXT.md D-05; RESEARCH.md Pattern 5.

### Joystick direction indicators (hooks refactor)

The four direction indicators (`up`, `down`, `left`, `right`) must animate identically after the refactor.

| Parameter | Value |
|-----------|-------|
| Active opacity | `1.0` |
| Inactive opacity | `0.25` |
| Active scale | `1.4` |
| Inactive scale | `1.0` |
| Transition driver | `activeDirection` SharedValue (existing) |

**Implementation pattern (D-11):** Each indicator gets its own top-level `useAnimatedStyle` call. The `createIndicatorStyle` factory function is removed entirely. See RESEARCH.md Pattern 3 for exact code.

## Interaction States

### Body-fill screen

| State | Visual |
|-------|--------|
| Loading / canvas not yet measured | Canvas renders at 0 width/height; no balls visible |
| Physics initializing | All `MAX_BALLS` Circle elements exist in Skia tree; `r=0` / `color='transparent'` — invisible |
| Balls spawned, falling | Balls appear above canvas top edge, fall with gravity `y=2.0`, bounce at bottom |
| Balls settled | Balls at rest inside body silhouette; RAF loop stopped (screen can dim) |

### Screen wake lock (BUG-01)

| State | Expected behavior |
|-------|------------------|
| User actively gesturing on joystick | OS keeps screen awake (native gesture dispatch — expected, correct) |
| Balls settling on body-fill screen | Screen may start to dim after balls settle (RAF stopped) |
| Balls fully settled, body-fill screen open | Screen dims normally per OS timeout — this is the success condition |
| User returns to home screen from body-fill | RAF fully cancelled; no background work preventing dimming |
