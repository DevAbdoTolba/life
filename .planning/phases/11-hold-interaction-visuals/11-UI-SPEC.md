Source: 11-CONTEXT.md D-02

### Quadrant Fill (VIS-02)
- Up quadrant (top half, above horizontal separator): `pillar.positiveColor` at 20% opacity
- Right quadrant (right half, right of vertical separator): `pillar.positiveColor` at 20% opacity
- Down quadrant (bottom half): `pillar.negativeColor` at 20% opacity
- Left quadrant (left half): `pillar.negativeColor` at 20% opacity
- Fill activates on `isHolding.value === 1` for the quadrant matching `activeDirection`
- Opacity driven by `isHolding` SharedValue: `withTiming(0.2, { duration: 100 })` on hold-start, `withTiming(0, { duration: 150 })` on release
- Fill must not obscure separator lines ‚Äî render fill layer below separator lines in z-order

Source: 11-CONTEXT.md D-04, D-05

## Interaction Contract

### Quadrant Separator Lines

- **Render method:** Two absolute-positioned `View` elements inside `outerRing` Animated.View, below all other layers
- **Vertical line:** `position: 'absolute'`, `width: StyleSheet.hairlineWidth`, `height: JOYSTICK_SIZE`, `left: JOYSTICK_SIZE / 2`, `top: 0`, `backgroundColor: colors.border`, `opacity: 0.3`
- **Horizontal line:** `position: 'absolute'`, `height: StyleSheet.hairlineWidth`, `width: JOYSTICK_SIZE`, `top: JOYSTICK_SIZE / 2`, `left: 0`, `backgroundColor: colors.border`, `opacity: 0.3`
- **Z-order from bottom:** separators (z:1) ‚Üí quadrant fill overlays (z:2) ‚Üí flashOverlay (z:5) ‚Üí knob (z:10)
- Lines are static (no animation) ‚Äî always visible at rest

### Section Fill on Hold

- **Render method:** Four `Animated.View` instances using `useAnimatedStyle`, each covering one quadrant via absolute positioning clipped to the circular boundary
- **Quadrant geometry:** Each fill covers a half-circle sector using `overflow: hidden` on a 50√ó100 container, or use clip via borderRadius on rectangular halves. Recommended approach: render two rectangular halves per axis (left/right split for vertical separator, top/bottom split for horizontal), then combine opacity logic to target individual quadrants
- **Simpler approach (preferred):** Render one `Animated.View` per quadrant using `StyleSheet.absoluteFillObject` but animate only the active quadrant's opacity. Each quadrant View covers the full outerRing but is not clipped ‚Äî opacity is the primary visual differentiator. This matches the existing `flashOverlay` pattern already in Joystick.tsx.
- **Animation:** `withTiming(0.2, { duration: 100, easing: Easing.out(Easing.ease) })` on hold-start; `withTiming(0, { duration: 150 })` on hold-end / gesture end
- **Color logic:** Reuse `directionValence` SharedValue ‚Äî value 0 = positiveColor, value 1 = negativeColor; interpolate with `interpolateColor` matching the existing `flashAnimatedStyle` pattern

### Haptic Feedback (VIS-04)

- **Call site:** `LongPress.onStart` handler in Joystick.tsx, inside the `runOnJS` callback boundary
- **API:** `Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Heavy)`
- **Timing:** Fires before `runOnJS(handleHoldStart)(dir)` ‚Äî haptic is the first response to hold-start
- **Haptic hierarchy (full):**
  - Light: Button press (`src/components/ui/Button.tsx`)
  - Medium: Quick swipe log (`useSwipeLog.ts`)
  - Heavy: Hold-start (this phase ‚Äî NEW)
  - Success notification: Target selected via radial menu (`useSwipeLog.ts`)

Source: 11-CONTEXT.md D-07, D-08

## Implementation Notes for Executor

1. Read `src/components/joystick/Joystick.tsx` before implementing ‚Äî understand existing SharedValue topology (`isHolding`, `activeDirection`, `directionValence`, `dragIntensity`).
2. Separator lines should be added as plain (non-animated) `View` elements ‚Äî no SharedValue needed. Place them as first children inside `outerRing` Animated.View.
3. Quadrant fill should follow the `flashOverlay` pattern already established in Joystick.tsx (`StyleSheet.absoluteFillObject`, `borderRadius: JOYSTICK_SIZE / 2`). Four overlays, each driven by a `useAnimatedStyle` comparing `activeDirection.value` and `isHolding.value`.
4. Do NOT grow the animated style count unnecessarily. Four quadrant fill styles + existing 8 styles = 12 total `useAnimatedStyle` calls ‚Äî within safe limits now that HOOK-01 is resolved.
5. Haptic call must be `runOnJS`-wrapped if called from within a worklet context, or placed directly in the JS-thread `handleHoldStart` callback.
6. Phase 11 does NOT implement app background color shift ‚Äî that is Phase 13 (VIS-05). Only the joystick quadrant fill is in scope here.
