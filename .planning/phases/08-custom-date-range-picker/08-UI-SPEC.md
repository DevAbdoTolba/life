## Interaction Contract

### CustomDateRangeModal — Two-Step Flow

**Entry:** Modal opens when user taps "Custom" period pill (or re-taps it while already selected).

**Step 1 — Start date:**
- Header: "Select Start Date"
- Inline `DatePicker` with `mode="date"` and `theme="dark"` — no `modal` prop (inline only, avoids modal-within-modal)
- No `minimumDate` constraint on start step
- Action button label: "Next" (accent background `#F5A623`, black text `#000`)
- Tapping "Next" advances `step` state to `'end'`

**Step 2 — End date:**
- Header: "Select End Date"
- Inline `DatePicker` with `mode="date"`, `theme="dark"`, and `minimumDate={tempStart}` — prevents end < start
- Action button label: "Confirm" (accent background `#F5A623`, black text `#000`)
- Tapping "Confirm" calls `onConfirm({ start: tempStart.toISOString(), end: tempEnd.toISOString() })` and closes modal

**Step indicator:** Two dots below the header — filled accent dot for current step, `colors.surfaceLight` dot for inactive step.

**Back navigation:** "Back" text button (left-aligned, `colors.textSecondary`) visible on step 2 only — returns to step 1 without resetting `tempStart`.

**Dismiss:** Tapping the overlay (`Pressable` with `StyleSheet.absoluteFill` behind sheet) closes modal without applying changes. Android back button also dismisses.

**Default values on open:** `tempStart` = first day of current month, `tempEnd` = today — matching `getPeriodDates('custom')` fallback.

**After confirm:** PeriodSelector subtitle updates to "Mar 1 – Mar 24" format via `formatDateRangeLabel`. Analytics `useEffect` fires with new `customRange`.

### PeriodSelector — Custom Pill Subtitle

When `selected === 'custom'` and `customLabel` prop is provided:
- Render a single `Text` line centered below the pill row
- Font: 12px (sm), Inter_400Regular, color `colors.textSecondary`
- Content: e.g., "Mar 1 – Mar 24"
- When `selected !== 'custom'`: subtitle is hidden (not rendered)

When user taps "Custom" pill while `selected === 'custom'`:
- `onSelect` fires with the same value — parent must also wire `onCustomPress` to re-open the modal
- Implementation: `onPress` calls `onSelect(period.key)` AND calls `onCustomPress?.()` when `period.key === 'custom'` and already selected

### Modal Presentation

- `animationType="slide"` — slides up from bottom (matches NoteEntryModal)
- `transparent={true}` — overlay backdrop renders behind sheet
- `onRequestClose` — closes on Android back button without confirming

## Phase-Specific Notes

**Native build requirement:** `react-native-date-picker` requires a native dev build. Do NOT test in Expo Go. Run `npx expo prebuild` after install to generate `ios/` and `android/` directories, then build with `npx expo run:ios` or `npx expo run:android`.

**Inline DatePicker only:** Use the inline variant (`onDateChange` prop, no `open` / `modal` prop) inside the custom bottom sheet. Using the `modal` prop creates a native modal on top of the RN Modal, causing visual conflicts.

**Dark theme:** `theme="dark"` prop on `DatePicker` handles all internal calendar cell styling. Do not attempt to override internal DatePicker styles — the native picker owns its internals.
