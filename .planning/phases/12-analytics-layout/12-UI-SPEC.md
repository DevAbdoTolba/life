## Spacing Scale

Declared values from `src/constants/theme.ts` (spacing object uses multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot-to-text gap in LogHistoryItem |
| sm | 8px | Internal row gaps, vertical list item padding |
| md | 12px | Horizontal list item padding, chart title margin-bottom |
| lg | 16px | Screen horizontal padding (via xl), section labels |
| xl | 24px | Screen-level horizontal padding (paddingHorizontal) |
| xxl | 32px | Header padding-bottom, triangle gap |
| xxxl | 48px | List bottom padding, joystick triangle vertical spacing |

Exceptions:
- Activity list peek height: 60px (one full LogHistoryItem row — hardcoded, not a token)
- LogHistoryItem note indent: 16px (aligns under direction text, equals dot 8px + gap 8px)
- Touch target minimum: 44px (RADIAL_BUBBLE_SIZE already meets this)

Source: `src/constants/theme.ts`, `src/components/joystick/constants.ts` (D-06 from CONTEXT.md)

## Chart Specification: Daily Activity Line Chart

Replaces `PillarBarChart`. All values are prescriptive, not exploratory.

| Property | Value | Rationale |
|----------|-------|-----------|
| Chart type | `LineChart` with `dataSet` prop (multi-line) | D-04, matches `TrendLineChart` pattern |
| Height | 180px | Matches existing `PillarBarChart` height — no container overflow |
| Width | `useWindowDimensions().width - spacing.xl * 2 - spacing.lg * 2` | Matches both chart components' width formula |
| Lines | 3 — one per pillar (Afterlife, Self, Others) | D-01 |
| Line thickness | 2px | Matches `TrendLineChart` existing value |
| Curve | `curved: true` | Matches `TrendLineChart` — shows trend continuity per requirement |
| Area fill | `areaChart: true`, startOpacity 0.15, endOpacity 0 | Matches `TrendLineChart` — visual depth without distraction |
| Data points | Hidden (no `showDots` — default off) | Keeps chart clean; trend line is the signal |
| X-axis labels | Day abbreviations (Sun/Mon/…) for week; date number for month; "Today" for today | Matches existing `getDayLabel` logic |
| Y-axis | `noOfSections: 4`, auto-scaled to max count (min 5) | Matches existing bar chart `maxValue` logic |
| Grid | `rulesType: "dashed"`, `rulesColor: colors.border` | Matches existing chart chrome |
| Background | `colors.surface` (`#14141F`) | Matches card background |
| Animation | `isAnimated: true` | Consistent with existing charts |
| Container | `Card` component with "Daily Activity" title at 16px semibold | Matches existing `PillarBarChart` structure |

Source: CONTEXT.md D-01–D-04, `TrendLineChart.tsx`, `PillarBarChart.tsx`

## Layout Specification: Home Screen

### Current Structure (to be modified)

`SafeAreaView` > `FlatList` with `ListHeaderComponent` containing header + joystick triangle. Activity list is `FlatList` data below the header.

### Target Structure

The joystick triangle stays in the lower portion of the viewport (already positioned there per D-05). The activity list peeks 60px below the triangle bottom edge.

#### Implementation Contract

| Element | Spec |
|---------|------|
| Screen layout | `SafeAreaView` (flex: 1) with `backgroundColor: colors.background` |
| Header | Compact — app name + action count. Reduces vertical space consumed above joysticks. `paddingTop: spacing.lg`, `paddingBottom: spacing.lg` (reduced from current `spacing.xxl`) |
| Header title | "Hayat" at 36px bold, `letterSpacing: -1` |
| Header subtitle | "{N} actions today" at 10px regular, `colors.textMuted` |
| Joystick triangle container | Positioned to consume remaining screen space above the peek strip. `flex: 1` wrapping the triangle keeps joysticks naturally in the lower half |
| Triangle layout | Unchanged — top row (Afterlife center) + bottom row (Self + Others, 85% width, space-between) |
| Activity list peek mechanism | `FlatList` with `style={{ maxHeight: 60 }}` at initial rest state, OR a fixed-height peek strip showing only the most recent `LogHistoryItem`. Mechanism choice: fixed 60px peek strip using `FlatList` with `initialNumToRender: 1` and scroll enabled. User scrolls up to see full list. |
| Peek height | 60px — fits one full `LogHistoryItem` row (paddingVertical 8px × 2 + direction row ~28px + border = ~52px, padded to 60px with container minHeight) |
| Peek border | `borderTopWidth: 1`, `borderTopColor: colors.border` — visual separator between joystick zone and list zone |
| Peek label | "Today's Activity" label removed from above the list. The peeking row is self-explanatory. If list is empty: see empty state below. |
| Scroll behavior | `FlatList` `scrollEnabled: true`, `showsVerticalScrollIndicator: false`. Scrolling up reveals full history. |
| List bottom padding | `paddingBottom: spacing.xxxl` (48px) |

Source: CONTEXT.md D-05–D-08, `app/(tabs)/index.tsx`, `src/components/ui/LogHistoryItem.tsx`

## Interaction States

### Daily Activity Line Chart

| State | Visual |
|-------|--------|
| Has data | Multi-line area chart, animated on mount |
| Empty | 180px container, centered text "No activity for this period" at 12px regular `colors.textMuted` |
| Loading | Not applicable — data is synchronous from local store |
| Overflow | Chart width formula prevents overflow: `width - 48px` always fits within `Card` padding |

### Activity List Peek

| State | Visual |
|-------|--------|
| Has logs today | Most recent `LogHistoryItem` visible in 60px strip; pillar color dot + direction + time visible |
| No logs today | 60px strip shows "No activity yet" centered at 12px regular `colors.textMuted` |
| Scrolling | List expands naturally; joystick triangle scrolls out of view when user pulls list up |
