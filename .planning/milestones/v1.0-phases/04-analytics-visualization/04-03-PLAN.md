Ball aggregation logic per ADR-026 and D-17:
```typescript
const MAX_BALLS = 50;

function aggregateLogs(logs: Log[], scaleX: number, scaleY: number): BallConfig[] {
  if (logs.length === 0) return [];
  const scaleFactor = logs.length > MAX_BALLS ? logs.length / MAX_BALLS : 1;
  const baseRadius = 6 * Math.min(scaleX, scaleY); // 6 points at 200x400 base, scaled
  const aggregatedRadius = baseRadius * Math.cbrt(scaleFactor);

  const displayLogs = logs.length > MAX_BALLS
    ? logs.filter((_, i) => i % Math.ceil(logs.length / MAX_BALLS) === 0).slice(0, MAX_BALLS)
    : logs;

  return displayLogs.map((log) => ({
    dropX: 40 * scaleX + Math.random() * (120 * scaleX), // drop within torso width
    radius: aggregatedRadius,
    color: getLogColor(log.pillarId as PillarId, log.direction as SwipeDirection),
  }));
}
```

Hook signature:
```typescript
export function useBodyFillPhysics(
  logs: Log[],
  canvasWidth: number,
  canvasHeight: number
): { ballStates: React.MutableRefObject<BallState[]>; isSettled: SharedValue<boolean> }
```

Implementation:
- Compute scale factors: `scaleX = canvasWidth / BODY_DIMENSIONS.width`, `scaleY = canvasHeight / BODY_DIMENSIONS.height`
- Create engine: `Matter.Engine.create({ gravity: { x: 0, y: 1.2 } })`
- Add static walls from `BODY_WALLS`, scaling each wall's x/y/width/height by scaleX/scaleY: `Matter.Bodies.rectangle(wall.x * scaleX, wall.y * scaleY, wall.width * scaleX, wall.height * scaleY, { isStatic: true })`
- Aggregate logs to balls with `aggregateLogs(logs, scaleX, scaleY)`
- Drop balls with staggered timing: `setTimeout(() => { ... }, i * 120)` per D-18 (chronological order)
- Each ball: `Matter.Bodies.circle(ball.dropX, -ball.radius, ball.radius, { restitution: 0.3, friction: 0.1, frictionAir: 0.02 })`
- Create `BallState` with `makeMutable(initialX)` and `makeMutable(initialY)` for each ball
- Game loop via `requestAnimationFrame`:
  ```typescript
  Matter.Engine.update(engine, 1000 / 60);
  bodiesRef.current.forEach((body, i) => {
    if (ballStates.current[i]) {
      ballStates.current[i].x.value = body.position.x;
      ballStates.current[i].y.value = body.position.y;
    }
  });
  ```
- Cleanup in useEffect return: `cancelAnimationFrame(rafRef.current)`, `Matter.World.clear(world, false)`, `Matter.Engine.clear(engine)` — per Research Pitfall 3
- Track `isSettled` (set to true when all ball velocities are < 0.5 for 30 consecutive frames)

Implementation:

- Recommended approach: Use `<Group transform={[{ scaleX: width / 200 }, { scaleY: height / 400 }]}` on the outer group, and keep ball positions in scaled coordinates from the physics hook.

Actually, since the physics hook already scales coordinates, just use the raw clip path and scale it:
```typescript
function getScaledBodyPath(width: number, height: number) {
  const path = Skia.Path.MakeFromSVGString(BODY_SVG_PATH);
  if (!path) return null;
  const scaleX = width / 200;
  const scaleY = height / 400;
  path.transform(Skia.Matrix().scale(scaleX, scaleY));
  return path;
}
```

Use `useMemo` to cache the scaled path based on width/height.

Render:
